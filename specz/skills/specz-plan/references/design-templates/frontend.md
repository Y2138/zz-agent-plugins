# Frontend Design Hints

Use only when the primary end is frontend.

Include when relevant:

- routes/pages/components/hooks/stores/client services/tests
- state owner, fetch/transform/render/submit/refresh path
- request/response fields, form fields, URL params, storage keys
- loading/empty/error/disabled/validation/navigation behavior
- accessibility/responsive behavior when acceptance depends on it

Avoid:

- backend internals unless they change the client contract
- generic styling/design-system reminders
